### Цели модуля
Данный модуль создает фундаментальное понимание технологии представления информации в веб-интерфейсах и готовит студентов к дальнейшему изучению динамических и интерактивных веб-приложений.

**Основные цели**
- Обеспечить студентов фундаментальными знаниями о языке разметки HTML и его элементах, необходимых для создания структурированных веб-страниц с корректным и семантически правильным содержанием.

- Научить использовать базовые инструменты оформления и форматирования контента с помощью CSS, что позволит создавать визуально привлекательные и удобные для пользователя интерфейсы.

- Развить понимание принципов организации интерфейсных элементов, таких как заголовки, списки, таблицы, гиперссылки и изображения, а также их грамотно применять в реальных веб-проектах.

- Познакомить с ограничениями и возможностями классических веб-технологий при создании интерактивных и адаптивных интерфейсов.

- Ознакомить с современными средами разработки и популярными фреймворками и библиотеками (React, Angular, Flutter), используемыми для ускорения и упрощения процесса создания пользовательских интерфейсов.

- Сформировать умения работать с современными инструментами для разработки UI, а также мотивировать к дальнейшему углубленному изучению динамического программирования на стороне клиента.

Таким образом, модуль направлен на формирование прочной базы веб-интерфейсного программирования и подготовку к дальнейшему освоению сложных технологий создания интерактивных и отзывчивых пользовательских интерфейсов в вебе.

### Язык разметки HTML: структура и основные элементы (15 ч)
Структура HTML-документа: корневой элемент `<html>`, секции `<head>` и `<body>`.

#### Основные элементы для контента (**визуальные элементы**):

- Заголовки `<h1>`–`<h6>` для структурирования текста и выделения разделов

- Абзацы `<p>`

- Списки: упорядоченные `<ol>`, неупорядоченные `<ul>`, элементы списка `<li>`

- Таблицы для табличных данных: `<table>`, строки `<tr>`, ячейки `<td>`, заголовки `<th>`

- Изображения `<img>`: атрибуты `src`, `alt`, размеры, атрибуты доступа

#### Интерактивные элементы

- Гиперссылки `<a>`: атрибуты `href`, `target`, навигация по сайту и внешним ресурсам

- Элементы форм: `<form>`, `<input>`, `<select>`, `<option>`, `<button>`

Интерактивные элементы HTML — это такие теги, которые предназначены для взаимодействия пользователя с веб-страницей, позволяющие вводить, отправлять и обрабатывать данные или переходить по ссылкам.

Основные интерактивные элементы и их назначение:
- `<a>` (якорь, ссылка)

    Позволяет переходить на другую страницу или участок текущей страницы. Атрибуты:

    - `href` — адрес ссылки

    - `target` — как открывать (новая вкладка, текущее окно)

    - `rel` — тип отношений с ссылкой (например, `nofollow`)

    - `download` — позволяет скачать файл по ссылке

- `<form>` (форма)

    Контейнер для элементов ввода и отправки данных на сервер. Атрибуты:

    - `action` — URL для отправки данных

    - `method` — метод отправки (GET, POST)

    - `enctype` — тип кодирования данных

    - `name` — имя формы

- `<input>` (поле ввода)

    Позволяет создавать разные виды элементов ввода: текстовые поля, кнопки, чекбоксы, радиокнопки, пароли и др. Тип задаётся атрибутом `type`. Атрибуты:

    - `type` (`text`, `password`, `checkbox`, `radio`, `submit` и т.д.)

    - `name` — имя элемента для передачи данных формы

    - `value` — значение элемента

    - `placeholder` — подсказка внутри поля

    - `required` — обязательное для заполнения

    - `disabled` — недоступно для взаимодействия

    - `readonly` — только чтение

- `<textarea>` (многострочное текстовое поле)

    Для ввода большого объёма текста.

- `<button>` (кнопка)

    Кнопка для отправки формы или запуска скриптов. Атрибут `type` может быть `submit`, `reset` или `button`.

- `<select>` и `<option>`

    Выпадающий список с вариантами выбора. `<option>` задаёт отдельный вариант.

- `<label>`

    Метка для элементов формы. Связывается с элементом через атрибут `for` или оборачиванием элемента, улучшая доступность.

**Особенности атрибутов интерактивных элементов**: атрибуты позволяют управлять поведением элементов, например, валидировать ввод (`pattern`, `min`, `max`), ограничивать доступ (`disabled`), задавать начальные значения и связывать элементы с данными и скриптами для динамического взаимодействия.

**Роль интерактивных элементов**: они делают веб-страницы динамичными, позволяют пользователям вводить и отправлять информацию, реализуют навигацию и взаимодействие, что является основой современного веб-приложения.

Краткий пример формы с интерактивными элементами:
```html
<form action="/submit" method="post">
  <label for="name">Имя:</label>
  <input type="text" id="name" name="username" placeholder="Введите имя" required>

  <label for="email">Email:</label>
  <input type="email" id="email" name="useremail" placeholder="Введите email" required>

  <label for="gender">Пол:</label>
  <select id="gender" name="usergender">
    <option value="male">Мужской</option>
    <option value="female">Женский</option>
  </select>

  <button type="submit">Отправить</button>
</form>
```

Эти элементы широко используются для создания форм обратной связи, регистрации, авторизации и других пользовательских интерфейсов с интерактивным вводом данных.

#### Структурные элементы

- Универсальные блочные элементы: `<div>`

- Семантические блочные элементы: `<header>`, `<footer>`, `<main>`, `<nav>`, `<article>`, `<section>`.

- Семантика и правильное использование элементов для повышения доступности страниц

Блок `<div>` — это универсальный блочный элемент в языке HTML, используемый для группирования и структурирования элементов веб-страницы. Он не имеет собственного семантического значения, а служит контейнером для объединения различных частей содержимого (текста, изображений, других тегов и интерфейсных элементов) с целью управления их отображением через стили CSS.

Как блочный элемент, `<div>` всегда занимает всю доступную ширину и начинается с новой строки. Это позволяет создавать логические блоки и секции на странице, например для разделения макета на шапку, контент, колонку и подвал. Использование `<div>` облегчает создание адаптивных и удобочитаемых макетов, а также стилизацию и позиционирование элементов на странице. При этом для придания визуального вида и позиционирования обычно применяют атрибуты `class` или `id` с CSS свойствами.

Важно отметить, что при частом и неосмысленном использовании `<div>` код может стать сложным, поэтому для семантической разметки предпочтительнее использовать специализированные теги, такие как `<section>`, `<article>`, `<header>` и т. п. Однако `<div`> остается основным строительным блоком верстки для многих случаев, где необходим простой контейнер без семантики.

Кратко о `<div>`:
- Универсальный блочный контейнер без семантики.

- Используется для группирования и структурирования элементов.

- Начинается с новой строки, занимает всю ширину родителя.

- Служит основой для стилизации через CSS.

- Применяется для создания макетов и разделения страницы на логические блоки.

- Не несет особого смыслового значения, используется в основном для оформления и структуры.

Если нужны более семантические разделы, лучше использовать специальные HTML5 теги, а `<div>` применять для случаев, где такого смысла не требуется

Семантические теги HTML5 — это специальные HTML-элементы, которые не только структурируют веб-страницу, но и несут смысловую нагрузку, помогая браузерам, поисковым системам и вспомогательным технологиям (например, экранным читалкам) лучше понимать содержание и его структуру. В отличие от универсального контейнера <div>, семантические теги дают ясное представление о роли блоков на странице.

Основные семантические теги HTML5 и их назначение:
- `<header>` — шапка страницы или раздела, обычно содержит заголовок, логотип, навигацию и другое вводное содержимое.

- `<nav>` — секция с навигационными ссылками по сайту или разделу.

- `<main>` — основное, уникальное содержимое страницы, единственный на странице.

- `<section>` — тематический раздел страницы, который группирует связанный контент.

- `<article>` — отдельный, самостоятельный материал, который может быть опубликован и использован отдельно (например, статья, новость, комментарий).

- `<aside>` — дополнительный контент, который связан с основным содержимым, например, боковая панель, ссылки, реклама.

- `<footer>` — нижняя часть страницы или раздела, обычно содержит контактную информацию, копирайт и ссылки.

- `<address>` — контактная информация автора или владельца сайта.

- `<figure>` и `<figcaption>` — для иллюстраций, фотографий или диаграмм, где `figcaption` служит подписью.

- `<time>` — для указания даты и времени, с атрибутом `datetime` для машинного чтения.

Зачем нужны семантические теги:
- Помогают поисковым системам лучше индексировать и ранжировать страницы.

- Повышают доступность сайта для пользователей с ограничениями (экранные читалки).

- Улучшают структуру документа и делают код более читаемым и поддерживаемым.

- Упрощают стилизацию и сценарное управление, поскольку ясно, к какому виду контента относится элемент.

Пример семантической структуры страницы:
```html
<header>
  <h1>Заголовок сайта</h1>
  <nav>Меню навигации</nav>
</header>
<main>
  <article>
    <h2>Статья</h2>
    <p>Текст статьи...</p>
  </article>
  <aside>
    <p>Боковая информация</p>
  </aside>
</main>
<footer>
  <address>Контакты</address>
</footer>
```

Использование семантических тегов — современный стандарт веб-разработки, позволяющий создавать логичные и удобные для всех участников экосистемы интернет-страницы.
